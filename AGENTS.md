# 会計データCSV分析リポジトリー

個人事業主の会計データ（「マネーフォワード クラウド確定申告」からエクスポートしたCSV）を年度別に格納し、分析を行う。

## データの配置

- 分析対象の元データとなる CSV は `data/{年度}/` 以下に UTF-8 で格納される（`.gitignore` 済み）。
- 元データはユーザーが GUI（`pnpm start`）経由でインポートする運用。
- Shift-JIS 形式の CSV は `*_sjis.csv` のようなファイル名で保存する。
  これはExcelで開く目的の形式であり、分析には UTF-8 形式のファイルを使用する。

## データの分析

### 基本フロー

1. `data/memo.txt` および `data/{年度}/memo.txt` で既知情報を確認
2. `uv run python -m checks.runner data/*/仕訳帳.csv` で全チェック実行
3. 警告を偽陽性と要確認に分類
4. 要確認の取引を `rg` で元データと照合
5. 発見事項・偽陽性をメモに記録

詳細な手順は [analysis-flow.md](analysis-flow.md) を参照。

### 方針

- 個別取引の深掘り調査など、テキスト検索には `rg`（ripgrep）を優先的に使う。
  CSVの摘要・取引先・科目の検索は `rg` で十分なことが多い。
  集計・計算など `rg` では難しい処理に限り、スクリプトを書く。
- スクリプトは最終成果物ではなく、AIが対話的に分析する際の手段の一つと考える。
  すべてを機械的に処理する必要はない。
- `data/{年度}/` 以下に、分析目的の中間集計データなどを保存してもよい。
  元データと区別するため、ファイル名に `derived_`, `tmp_` などのプレフィックスを付けること。

## CSV スキーマ定義

対応するCSV種別のスキーマ（カラム名、保存ファイル名、日付カラム）は `schema/` にJSONで定義されている。
Python（`checks/*.py`）と GUI（`gui/**/*.ts`）の両方がこのJSONを正本として参照する。

詳細は [schema/README.md](schema/README.md) を参照。

## チェックスクリプトの開発

- 有用なスクリプトは再利用可能な形で `checks/` に追加する。既存スクリプトを改善してもよい。
- スクリプトを新規作成・変更した場合は、対応するテストを `tests/checks/` に追加・更新すること。
- `checks/` のスクリプトの各関数は `CheckResult`（`warnings: int`, `skipped: bool`, `reason: str`）を返す。テストでは出力内容だけでなく戻り値も検証する。
- 追加手順・構成の詳細は [checks/README.md](checks/README.md) を参照。

## コマンド体系

### データ分析コマンド

Python の実行は `uv run python` に統一する（`uv` が仮想環境と依存を管理しているため）。

- **データ探索**: `rg`, `grep`, `cat`, `head`, `tail`, `wc`, `ls`
- **チェック実行・Python**: `uv run python ...`（ランナー、個別スクリプト、即興スクリプト）
- **テスト**: `uv run pytest`

### GUI開発コマンド

- **型チェック**: `pnpm typecheck`
- **テスト**: `pnpm test`（= `vitest run`）
- **ビルド**: `pnpm build`

## 会計の前提

- 通貨: 円（小数なし）
- 消費税: インボイス制度対応
- 会計期間: 1月1日〜12月31日
